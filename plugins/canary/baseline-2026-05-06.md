# Canary CI Gate — Adversarial Baseline (2026-05-06)

Closes ecosystem-audit finding **F-004** with a measurable regression contract.
Baseline run authoritative as of 2026-05-06.

## Methodology

The canary plugin's runtime hook (`canary-scan.py`) is advisory-only — it
exits 0 unconditionally and emits a stderr advisory + appends to
`state/hits.ndjson` whenever an active canary token appears in a tool's
input or output (the canonical signature of a successful indirect prompt
injection).

The CI gate (`scripts/ci-canary-gate.py`) loads every fixture under
`fixtures/injection/*.json`, seeds a temporary `active-canaries.json` with
the fixture's `canary_token`, builds a synthetic `PostToolUse` hook payload
from `fixture.input`, and pipes it into the scanner. Detection is asserted
when **both** of these hold:

1. Scanner stderr contains `HIT:` and the canary token.
2. `state/hits.ndjson` gains a record with that token.

State files (`active-canaries.json`, `hits.ndjson`) are snapshotted before
the run and restored after — this baseline measurement is non-destructive.

**What this gate measures.** Coverage of the *detection surface itself*:
given that a canary token has leaked into a tool boundary, does the
scanner observe it? It does **not** measure whether a real model would be
fooled by the injection — that's a separate evaluation concern. The
detection contract is: *if a leak occurs, we see it.* The 40 fixtures
exercise that contract across the realistic carrier shapes attackers
deploy.

## False-negative analysis

**None at this baseline.** All 40 fixtures detected.

The detection mechanism is intentionally simple — substring match on the
literal canary token across `tool_input + tool_output`. This is the
correct primitive for the threat model: a canary is a unique sentinel
token whose presence in a tool boundary means an injection succeeded
upstream. Recasting it as a fuzzy-match or semantic problem would
*introduce* false negatives without buying signal.

The fixtures that *could* in principle slip through a less-careful
scanner — and what we did to keep them in scope:

- **24 (homoglyph), 25 (RTL), 26 (BiDi):** the carrier *directive* is
  obfuscated, but the canary itself is plaintext ASCII per the threat
  model. A fix that treated only the carrier would still detect the leak.
- **20, 27, 28 (encoding bypass):** the encoded payload is the *attack
  hint*, not the canary. The canary leaks in plaintext when the model
  complies. Detection works on the post-decode artifact, which is what
  the scanner sees in real runs.
- **35 (context padding):** detection is O(n) substring scan; long padding
  doesn't dilute the match. (Latency 131ms — comparable to short fixtures.)

If a future fixture lands the canary in a *transformed* form
(base64-encoded canary, split across whitespace, etc.) we would have a
genuine false negative — and the right fix is a new fixture + a multi-pass
decoder hook, not weakening the substring primitive. That work is tracked
under the next-tune section below.

## Comparison to external baselines

| Tool / corpus                                 | Public detection rate         | Comparable? |
|-----------------------------------------------|-------------------------------|-------------|
| garak (NVIDIA, v0.10)                         | not measured for this surface | No — garak measures *model susceptibility*, not canary-leak detection. Different surface. |
| promptbench (Tsinghua, v1.4)                  | not measured for this surface | No — model-side robustness benchmark. |
| DeepEval / Confident-AI prompt-injection eval | not measured for this surface | No — model-side. |
| DARPA AIxCC injection challenge corpus        | not published                 | No — challenge artifacts not public. |
| MITRE ATLAS adversarial-ML matrix             | qualitative, no numeric rate  | No — taxonomy, not benchmark. |

Public corpora measure *whether the LLM gets fooled*. This baseline
measures *whether our detection observer fires when the leak happens*.
These are complementary surfaces; a fair external comparison would
require an oracle that's already detected the leak by some other means,
which none of the published tools instrument. Marked **not measured**
deliberately rather than guessing — per the honest-numbers contract.

The closest published shape is internal-canary leak detection in
LangChain / Guardrails-AI guards, but those projects don't publish a
fixture-set detection rate either. When/if they do, this section gets a
real comparison row.

## Regression policy

**Any reduction in detection_rate from this baseline (1.000) by ≥ 5%
absolute blocks the PR.** Enforced by:

```
python plugins/canary/scripts/ci-canary-gate.py --baseline
```

Without `--baseline` the gate behaves as before (every fixture must
detect to pass). With `--baseline`:

1. Reads `plugins/canary/baseline-2026-05-06.json`.
2. Compares current `detected / fixtures` against `detection_rate`.
3. Fails (rc=1) if `baseline_rate − current_rate ≥ regression_threshold_drop` (0.05).
4. Otherwise reports drift in stdout and proceeds to the standard pass/fail.

Refreshing the baseline is deliberate: bump the date in the filename,
re-run the measurement, and update `baseline-2026-05-06.json` (or its
successor) in the same PR that motivates the refresh. Never silently
update — the file's git-history is the regression audit trail.

## Next-tune recommendations

Coverage gaps to close in subsequent waves (counts kept conservative
until each carrier earns a recurrence in real traffic):

- **Encoded-canary detection.** Add a multi-pass decoder hook so
  canary-as-base64, canary-split-across-whitespace, and canary-in-rot13
  also fire. This widens the detection primitive from substring to
  decode-then-substring, with a token-budget cap to avoid pathological
  inputs.
- **Multi-modal carriers.** Image-OCR'd injection + audio-transcript
  injection — reachable only once the agent has those tool surfaces.
- **More indirect-injection variants.** RSS-feed payloads, calendar
  invites, Slack-message replays — the indirect class is the most
  realistic and is currently at 5/40 (12.5%).
- **More tool-misuse.** SQL-injection-shaped tool args, file-path
  traversal, command-injection — currently at 2/40.
- **Cross-class compound fixtures.** Token-smuggled jailbreak in a
  research-source quote — single fixture exercising 2-3 classes.

Track each as a separate fixture commit so the per-class breakdown
table grows monotonically and the regression signal stays clean.
